# Privacy Frameworks Reference

## Regulatory Frameworks

### HIPAA (Health Insurance Portability and Accountability Act)
- **Scope**: US healthcare data
- **Key Principles**: Minimum necessary, safeguards, patient rights
- **Clinical Trial Relevance**: Direct — governs how trial data is handled
- **Web Security Lesson**: Apply "minimum necessary" principle to all data access

### GDPR (General Data Protection Regulation)
- **Scope**: EU personal data
- **Key Principles**: Lawful basis, purpose limitation, data minimization, accuracy, storage limitation, integrity/confidentiality, accountability
- **Clinical Trial Relevance**: EURACAN registry explicitly cites GDPR compliance
- **Web Security Lesson**: Build privacy principles into system design from day one

### CCPA/CPRA (California Consumer Privacy Act)
- **Scope**: California residents' data
- **Key Principles**: Right to know, delete, opt-out, non-discrimination
- **Web Security Lesson**: Implement user-facing data controls and APIs

## Technical Standards

### NIST Privacy Framework
- **Identify**: Understand data flows and privacy risks
- **Govern**: Establish privacy policies and processes
- **Control**: Implement technical and organizational measures
- **Communicate**: Transparent privacy practices
- **Protect**: Monitor and respond to privacy events

### ISO 27701 (Privacy Information Management)
- Extension of ISO 27001 for privacy
- PIMS-specific controls and guidance
- Maps to GDPR requirements

## Implementation Checklist

- [ ] Data inventory and classification complete
- [ ] Privacy impact assessments conducted
- [ ] Consent management system in place
- [ ] Data subject access request (DSAR) process defined
- [ ] Encryption at rest and in transit implemented
- [ ] Access control policies reviewed quarterly
- [ ] Audit logging comprehensive and reviewed
- [ ] Breach notification procedure tested
- [ ] Third-party data processing agreements in place
- [ ] Privacy policy reviewed and updated
